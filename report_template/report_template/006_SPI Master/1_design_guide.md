# SPI Master回路およびテストベンチ説明書

## 対象ファイル
- `spi_master.v`: SPIマスター回路
- `tb_spi_master`: 検証用テストベンチ

## 回路概要

本回路は、SPI通信におけるマスター側の送受信回路である。

SPI通信では、マスターが通信全体を制御する。
本回路では、start 入力を受けるとスレーブ選択信号 ss を有効化し、SPIクロック sck を生成しながら、送信データ tdat を dout から1bitずつ出力する。
同時に、スレーブ側から返ってくるデータを din から1bitずつ読み取り、8bit分の受信が完了すると rdata に格納する。

SPI通信は、日常生活で使用されている多くの電子機器の内部で利用されている。例えば、マイコンと温度センサや加速度センサとの通信、液晶ディスプレイへの表示データの送信、フラッシュメモリとのデータの読み書きなどに用いられている。スマートフォンや家電製品、IoT機器などの内部でも広く使われており、本回路のようなSPIマスターがこれらのデバイスとのデータのやり取りを担っている。

## 実装する通信仕様の概要
本回路で扱う SPI 通信は、クロック信号に同期して1bitずつデータを送受信する同期式シリアル通信である。

SPI通信では、マスターが sck を出力し、そのクロックに合わせてデータの送受信を行う。

SPI通信で主に使用される信号は以下のとおりである。

|一般的なSPI信号 | 本回路での信号名 | 役割 |
| --- | --- | --- |
|SCK | sck | マスターが生成するSPIクロック |
| MOSI | dout | マスターからスレーブへ送るデータ線 |
| MISO | din | スレーブからマスターへ返すデータ線 |
| SS / CS | ss | 通信対象のスレーブを選択する信号 |

本回路では、ss は active low の信号である。
そのため、通信していない待機状態では ss=1 となり、通信中は ss=0 となる。

また、本回路は SPI mode 3 の動作を行う。
SPI mode 3 では、待機時の sck は High であり、送信データは sck の立下りで変化し、受信データは sck の立上りで読み取られる。

SPI mode 3:
- 待機時の sck: High
- dout の更新: sck の立下り
- din の読取り: sck の立上り

今回のテストベンチでは、テストベンチ側が擬似的なスレーブの役割を持つ。
テストベンチは、start を1クロックだけ 1 にして通信開始を指示し、tdat に送信データを与える。
また、din に受信させたいビット列を与えることで、SPIマスターが受信した結果として rdata が期待値になるかを確認する。

本回路では、送受信するビット順を mlb によって切り替えることができる。

mlb	ビット順
0	LSB first
1	MSB first

LSB first では、8bitデータの下位ビットから順に送受信する。
MSB first では、8bitデータの上位ビットから順に送受信する。

また、cdiv によって sck の分周比を切り替えることができる。

cdiv	sck の分周
2'b00	clk/4
2'b01	clk/8
2'b10	clk/16
2'b11	clk/32

今回の検証では、cdiv=2'b00 と cdiv=2'b01 を比較し、sck の周期が変化することを確認する。

## `spi_master.v`
### 入力信号
- `rstb`: active low の非同期リセット信号
- `clk`: 回路全体の基準クロック
- `mlb`: 送受信するビット順を指定する信号
- `start`: SPI通信の開始を指示する信号
- `tdat[7:0]`: マスターから送信するデータ
- `cdiv[1:0]`: SPIクロック sck の分周比を指定する信号
- `din`: スレーブ側からマスターへ返ってくる受信データ入力

### 出力信号
- `ss`: スレーブ選択信号
- `sck`: SPIクロック
- `dout`: マスターからスレーブへ送信するデータ
- `done`: 8bit送受信の完了を示す信号
- `rdata[7:0]`: din から受信した8bitデータ

### 内部レジスタ
- `cur`: 現在の状態を保持する状態レジスタ
- `nxt`: 次状態を示す信号
- `treg[7:0]`: 送信データを保持し、dout へ順に出力するためのレジスタ
- `rreg[7:0]`: din から読み取った受信データを一時的に保持するレジスタ
- `nbit[3:0]`: 現在何bit目まで送受信したかを数えるカウンタ
- `mid[4:0]`: cdiv に応じた sck の分周値
- `cnt[4:0]`: sck を生成するためのカウンタ
- `shift`: 送受信動作中であることを示す制御信号
- `clr`: 送受信関連の内部レジスタを初期化する制御信号

### 機能
- `start` 入力により SPI 通信を開始する
- 通信中は `ss=0` としてスレーブを選択する
- `cdiv` の設定に従って SPIクロック `sck` を生成する
- `tdat` の値を `dout` から1bitずつ送信する
- `din` から1bitずつデータを受信する
- 8bit分の受信データを `rdata` に格納する
- 8bit送受信が完了すると `done=1` とする
- 通信完了後に `ss=1`、`sck=1` の待機状態へ戻る

### 主要ステータス信号とテスト内容
#### `ss` の動作確認
意味:
- `ss`は通信対象のスレーブを選択する信号である．
- 本回路ではactive low のため、通信中は `ss=0` となる。
- 待機状態および通信完了後は ss=1 となる。

テスト内容:
- `RESET`
  - リセット後に ss=1 となることを確認する。
- `CASE1`から`CASE4`
  - start 入力後に ss=0 となることを確認する。
  - 通信完了後に ss=1 へ戻ることを確認する。

#### `sck` の動作確認
意味:
- sck は、SPIマスターが生成するSPIクロックである。
- SPI mode 3 のため、待機時は sck=1 となる。
- 通信中は cdiv の設定に従ってトグルする。

テスト内容:
- `RESET`
  - リセット後に `sck=1` となることを確認する。
- `CASE1`
  - `cdiv=2'b00` のとき、sck周期が `40 ns` となることを確認する。
- `CASE3`
  - `cdiv=2'b01` のとき、sck周期が `80 ns` となることを確認する。
  - `cdiv` の変更によって `sck` の周期が変化することを確認する。
- 各ケース共通
  - 通信完了後に `sck=1` へ戻ることを確認する。
